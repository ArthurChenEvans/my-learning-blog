On Flexbox

Created: 31st July 2025 | Last Updated: 4th August 2025

Flexbox Basics

A flexbox is simply a type of container that allows for more granular position of its child items using an x/y-axis for alignment. On top of item alignment, it also takes care of spacing, gaps, item size distribution based on available space, and how items wrap around based on available space.

Take for instance the following: 


.container {
    display: flex;
    justify-content: center;
    align-items: center;
}

The above sets up a normal box as a flexbox. The other two properties define how the items inside the box are aligned between two axes: main and cross, with justify-content handling the main-axis and align-items handling the cross-axis.

So, what is the main-axis and the cross-axis? The main-axis is the line that runs parallel to the flexbox's main direction, which there are two of: row and column. By default, row is the direction of the flexbox. In this case, the main-axis is the x-axis. So then, what is the cross-axis? It's the one that "crosses" the main-axis, or in other words, runs perpendicular to the flexbox's direction of flow. Using row's case again, this would mean the cross-axis is the y-axis. What do you think happens when we set the flex-direction property to column? If you guessed that the main-axis and cross-axis gets flipped, then you are correct! Remember, the main-axis will run parallel to the direction of the flexbox's flow, and the cross-axis will run perpendicular to it.

Note that these properties and all the ones that will come before flex sizing are placed on the parent container and NOT the individual child items.

Flex Flow

As stated prior, we can define the main and cross axes flow using the flex-direction property. From there, you can use justify-content to define how items are positioned along the main-axis and align-items for item positioning along the cross-axis. For specific values for these properties you can check the css tricks website here.

With the above being understood, you can now use another handy property that directs how items flow onto the next line (or not!). This is called the wrapping of a flexbox and uses the property flex-wrap to manage it. It's really simple as all it does to define whether items will resize themselves to fit on a line via flex-wrap: nowrap or will keep their size and move onto the next line via flex-wrap: wrap.

Lastly, there is another property to introduce for this section. However, this property doesn't do anything new but merges two properties we have already talked about together. This property is called flex-flow and takes two values: flex-direction and flex-wrap. Essentially, put in the values you'd usually put for those two properties into flex-fow in that stated order. For example:


.container {
    flex-flow: row wrap;
}

Flex Sizing

This section I will introduce three new flex properties: grow, shrink, basis. One thing to keep in mind here is that all 3 of these properties are put on the child items themselves and NOT the parent.

If it isn't already obvious based on the previous naming schemes, these three properties are named flex-grow, flex-shrink, and flex-basis respectively. For shorthand, you can use the flex property, which defines those three values in the aforementioned order. The reason I bring this up now is to show a common and default layout for the flex property:


.item {
    flex: 0 1 auto;
}

Firstly, I want to ask: what do you think this does? Recall that the property is applied on the child element. If you don't have any clue, that's fine â€” it's just an exercise to take a guess as well as to promote the hypercorrection effect*.

I also want you to keep in mind that for both flex-grow and flex-shrink, their values CANNOT be negative.

To begin with, We will start with the flex-grow part of the property. As we see, it is set to 0. If you've played around with this setting, then you'll notice an item with a flex-grow: 0 will not take up any leftover space within a container's line. Which makes sense considering the value is set to 0. If we were to set this value to say, 1, but all the other items in the container are still 0, then what do you think will happen? If you've guessed that only the item with a flex-grow: 1 will grow to fit the remaining space, then you'd be right! Let's try another thought experiment. Imagine all items within a container are set to flex-grow: 1. What do you think will happen now? Well, the answer is all of them will attempt to fill in any remaining space with said space being distributed equally (since they all share the same growth factor) among the items. Lastly, for extra brownie points, what do you think will happen if all child items have a flex-grow: 1 except for a lone child having a flex-grow: 2? Let's think it through: We now know that any number greater than zero will cause the item to attempt to fill in remaining space. So, if one of them has a larger growth factor than the others, it will receive a proportionally larger share of the leftover space. In this case it would be receiving twice as much of the extra space as the items with flex-grow: 1.

Now that the flex-grow property has been grasped, what do you think flex-shrink does? If you've guessed the opposite of flex-grow, then you'd be correct again! To put it simply, when the space of the container is smaller than the total size of all child items, then how much should I shrink by? Let's take a look at the default value set in flex. As you can see, it is set to 1. What this means is, by default, all child items within a flexbox container should shrink equally by cutting off occupied space from the children, where how much a child is cut by is relative to its flex-basis. The bigger the basis, the greater the cut. Additionally, by setting the flex-shrink to 0 you are essentially telling the child to not take part in the shrink calculation, which would cause overflow over the container.

Lastly, there is the flex-basis property with a default value of auto. This property defines the starting size before remaining space is distributed. The starting size of auto tells the property to check the child element's height or width property. Again, you can check css-tricks for more information regarding the available values; auto is just the most commonly used one. That being said, there is a special value used for flex-basis, especially when you need to ensure all items within a container are equally sized regardless of their content. To do this, we use a value of 0. What does that value do? Well, if we imagine a value of auto translating to "When growth happens, add onto my initial size the distributed space. When shrink happens, based on my initial size, cut off space from me", then we can think of setting a flex-basis to 0 as "When growth happens my size will be exactly the given distributed space. When shrink happens my size will always be my minimum width, even if that means overflow". Shorthand for this is typically flex: 1 1 0 or simply just flex: 1. Of course, this would also mean you will need to make ALL elements that value, or else they won't be sized equally.

Notes

Hypercorrection Effect: A psychological term to refer to the idea that correcting yourself after getting something you were confident about wrong improves your memory of the item compared to just getting the answer correct in the first place.
