Created: 26th August 2025 | Last Updated: 28th August 2025

Spread operator and Rest Pattern

The spread operator is a way to "unravel" arrays and objects into individual elements. To do so, you place ... in front of the to-be unraveled object or array like so: ...my_object. This operator is especially useful in two things: deconstructing and the rest pattern.

We'll start with deconstructing. Imagine you have an array of values you wish to put into their own variables. To do this in one line, you can wrap the variable names in braces and equate to the ...array like so: 


const person_array = [ 23, "Chen", 180 ];
const { age, name, height } = ...person_array

This can also be done for objects like so (the variable names must be the exact same as object's property names):


const person_object = { age = 23, name = "Chen", height = 180 };
const { age, name, height } = ...person_object

Next up is the rest pattern, which is putting the spread operator just before an argument in a function to say "pass the rest of the arguments in as a list." To demonstrate:


function sum(...numbers) {
    let total = 0;
    for (let number of numbers) {
        total += number;
    }
    return total;
}
sum(1, 2, 3, 4, 5); // 15

Logical Operators

There are three main logical operators in JS as well as their assignment variants: ||, &&, and ??. To start with the simplest one, || returns the first truthy value it finds, and if there are only falsy values, then it returns the last operand. && is similar, but instead returns the first falsy value it finds, but if both values are truthy, then it also returns the last operand. ?? returns the first non null or undefined value, and if there is none, then also returns the last operand.

These operators also have assignment variants: &&=, ||=, and ??=. They both assign the right operand based on the left variable's evaluation. &&= checks whether the left is truthy, ||= checks whether the left is falsy, and ??= checks whether the left is nullish. Now, regarding the nullish assignment operator, there is an extra variant used for chaining denoted with ?. or the optional chaining operator. By connecting values like so: user?.username which returns undefined if user is nullish, which helps in cases where you don't want to error on an empty object. To be precise, especially in longer chains, will return as soon as at least one of the chained properties is nullish.
